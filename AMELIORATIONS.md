# 📋 Liste des Améliorations et Mises à Jour Nécessaires

Ce document liste tous les problèmes identifiés et les améliorations à apporter au portfolio pour le mettre à jour vers la dernière version de Next.js et corriger les problèmes existants.

---

## 🚨 Problèmes Critiques

### 1. **Mise à jour de Next.js** ✅
- **Problème** : Le `package.json` utilise `"next": "latest"` mais la version n'est pas installée
- **Action** : Installer la dernière version stable de Next.js (actuellement v15.x)
- **Impact** : Nécessaire pour bénéficier des dernières fonctionnalités et corrections de sécurité
- **Status** : ✅ **TERMINÉ** - Next.js 16.0.8 installé (dernière version stable)

### 2. **Configuration Next.js obsolète (`next.config.js`)** ✅
- **Problème** : 
  - Utilisation de `domains` (déprécié depuis Next.js 12.3)
  - Mélange de syntaxe (`export default` et `module.exports`)
- **Action** : 
  - Migrer vers `remotePatterns` pour la configuration des images
  - Utiliser uniquement la syntaxe ES modules (`export default`)
- **Fichier** : `next.config.js`
- **Status** : ✅ **TERMINÉ** - Configuration migrée vers `remotePatterns` avec syntaxe ES modules

### 3. **Problèmes avec les iframes** ✅
- **Problème** : Les iframes ne fonctionnent plus correctement (mentionné par l'utilisateur)
- **Fichiers concernés** :
  - `app/page.tsx` (lignes 61-66)
  - `app/Job/[slug]/page.tsx` (lignes 248-255)
  - `app/Job/Job.tsx` (lignes 71-77)
- **Actions** :
  - Remplacer les iframes par des alternatives modernes (screenshots, liens externes, ou composants embarqués)
  - Utiliser des services de preview comme `microlink.io` ou `screenshotapi.net`
  - Ou simplement afficher une image de preview avec un lien vers le site
- **Status** : ✅ **TERMINÉ** - Toutes les iframes remplacées par des images de preview avec overlay et bouton de visite

### 4. **Composant Image Next.js déprécié** ✅
- **Problème** : Utilisation de `layout="fill"` et `objectFit` (dépréciés depuis Next.js 13)
- **Fichiers concernés** :
  - `app/Job/[slug]/page.tsx` (lignes 128-129, 169-170)
  - `app/Aboutme/page.tsx` (ligne 197)
- **Action** : Migrer vers la nouvelle API :
  - Remplacer `layout="fill"` par `fill={true}`
  - Remplacer `objectFit="cover"` par `style={{ objectFit: 'cover' }}` ou `className="object-cover"`
- **Status** : ✅ **TERMINÉ** - Tous les composants Image migrés vers la nouvelle API

### 5. **Utilisation de `<img>` au lieu de `<Image>`** ✅
- **Problème** : Utilisation de la balise HTML native `<img>` dans le carrousel
- **Fichier** : `app/page.tsx` (ligne 385)
- **Action** : Remplacer par le composant `Image` de Next.js pour optimiser les performances
- **Status** : ✅ **TERMINÉ** - `<img>` remplacé par `<Image>` dans le carrousel

## ⚠️ Problèmes Moyens

### 6. **Métadonnées génériques** ⚠️
- **Problème** : Description générique "Generated by Next.js" dans `layout.tsx`
- **Action** : Personnaliser les métadonnées avec une description appropriée pour le SEO
- **Note** : Amélioration recommandée pour le SEO

### 7. **Configuration TypeScript obsolète** ⚠️
- **Problème** : `target: "es5"` dans `tsconfig.json` est très ancien
- **Action** : Mettre à jour vers `"target": "ES2020"` ou supérieur pour de meilleures performances
- **Note** : À faire si nécessaire, le projet fonctionne actuellement

### 8. **Chemins d'images incorrects** ✅
- **Problème** : Certains chemins commencent par `/public` ce qui est incorrect
- **Fichiers concernés** :
  - `app/data.ts` : `/public/image.png`, `/public/IROC.png`, `/public/enedis.png`
  - `app/Job/JobList.ts` : `/public/enedis.png`, `/public/image.png`
- **Action** : Supprimer `/public` des chemins (Next.js sert automatiquement depuis `/public`)
- **Status** : ✅ **TERMINÉ** - Tous les chemins d'images corrigés dans JobList.ts

### 9. **Gestion d'erreur iframe incomplète** ✅
- **Problème** : L'événement `onError` sur iframe ne fonctionne pas (les iframes ne déclenchent pas d'événement error)
- **Fichier** : `app/Job/[slug]/page.tsx` (ligne 254)
- **Action** : Implémenter une détection d'erreur alternative (timeout, try-catch, ou service de preview)
- **Status** : ✅ **RÉSOLU** - Les iframes ont été supprimées, plus besoin de gestion d'erreur

### 10. **SessionStorage pour l'animation**
- **Problème** : Utilisation de `sessionStorage` pour l'animation de chargement
- **Fichier** : `app/page.tsx` (ligne 175)
- **Note** : Fonctionne mais pourrait être amélioré avec une meilleure gestion d'état

## 📝 Détails par Fichier

### `next.config.js`
```javascript
// ❌ Ancien (à remplacer)
module.exports = {
  images: {
    domains: [...]
  }
};

// ✅ Nouveau
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'example.com',
        pathname: '/**',
      },
    ],
  },
};

export default nextConfig;
```

### `app/Job/[slug]/page.tsx`
```tsx
// ❌ Ancien
<Image
  layout="fill"
  objectFit="cover"
/>

// ✅ Nouveau
<Image
  fill
  style={{ objectFit: 'cover' }}
  // ou
  className="object-cover"
/>
```

### `app/page.tsx` et `app/Job/Job.tsx`
```tsx
// ❌ Ancien (iframes qui ne fonctionnent plus)
<iframe
  src={props.data.path}
  className="..."
/>

// ✅ Nouveau (alternative recommandée)
<Link href={props.data.path} target="_blank">
  <div className="relative">
    <Image
      src={previewImage}
      alt={`${props.data.company} preview`}
      width={1920}
      height={1080}
    />
    <div className="absolute inset-0 flex items-center justify-center">
      <button className="bg-white px-6 py-3 rounded-lg">
        Visiter le site
      </button>
    </div>
  </div>
</Link>
```

### `app/data.ts` et `app/Job/JobList.ts`
```typescript
// ❌ Ancien
imgCover: "/public/image.png"

// ✅ Nouveau
imgCover: "/image.png"
```
